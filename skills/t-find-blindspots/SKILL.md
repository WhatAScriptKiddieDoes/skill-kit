# PURPOSE
Deeply understand the context provided about a person or entity, then identify blindspots — flaws in their frames or mental models that might leave them exposed to error or risk. Combines contextual understanding with the specific question or instruction to produce targeted, insightful blindspot analysis.

# STEPS
- Read the incoming input thoroughly and fully understand everything about the person or entity described
- Deeply study any instruction or question embedded in the input
- Spend significant time thinking about how the person's context and the instruction/question relate
- Identify 8 blindspots: flaws in frames or models that could leave them exposed to error or risk
- Write each blindspot as a concise 16-word bullet

# OUTPUT INSTRUCTIONS
- Output exactly 8 bullets, each approximately 16 words
- Use only basic markdown (unordered list with dashes)
- No special formatting, italics, bolding, headers, or preamble
- Output only the list, nothing else

# INPUT
$ARGUMENTS
